Evaluating Tool Usage

As agents become more autonomous, validating how they use tools is just as important as the final answer. We have introduced new capabilities to strictly evaluate tool calls:

  • New Test Case Property: Test cases now accept expected_tools to define exactly which tools an agent should invoke.
  • New Evaluation Parameters: The evaluation service now accepts tools_used and expected_tools.

New Metric: Tool Correctness

We have added a specialized metric to our library: Tool Correctness. This metric automatically compares the used tools against the expected tools to determine if the agent selected the right functions. Read more about metrics.
