Specification Analytics, CSV Export, and Expanded Documentation

2026-03-23

Specifications in Analytics

The analytics dashboard now includes a dedicated Specifications section that aggregates evaluation scores per specification, giving you a clear picture of how each policy area is performing across versions. Specification-level radar charts let you compare coverage at a glance, and clicking any specification label filters the entire dashboard to show only metrics linked to that specification.

Version comparison charts have also been improved with click-to-highlight and double-click-to-navigate directly to the evaluations list.

Test Case CSV Export

You can now export your test cases as CSV files directly from the dashboard. Select specific test cases or export all visible ones — the platform generates a comprehensive CSV and provides a direct download link. Use exported data for offline review, spreadsheet analysis, or integration with external workflows.

Documentation and Video Guides

The documentation has been significantly expanded with new video walkthroughs and reference pages:

  • Video guides now cover specifications, versions, traces, human evaluation, and AI-powered specification generation — each embedded directly in the relevant docs page.
  • A new Specifications tutorial walks you through the full specification workflow from creation to metric linking.
  • New reference pages for evaluation types, evaluation parameters, and endpoint connection configuration.

Platform Improvements

  • JSON field matching metric: A new deterministic metric that compares top-level JSON fields between actual and expected output, returning the ratio of matching fields as the score.
  • Endpoint connection validation: Before running an evaluation, the platform now validates that all required endpoint connections still exist — preventing runtime failures from deleted connections.
  • Credit warning badge: A visual indicator in the sidebar shows your organization's credit status — amber when credits are running low, red when exhausted.
  • Retry defaults: New endpoint connections default to retries enabled with 3 attempts and exponential backoff.
  • Form and table polish: Clearer validation errors, improved column widths, better form resets, and more consistent UI across the dashboard.

Coming Next

We are already working on the next set of features:

  • Trigger-Based Evaluations: Support for event-driven agents that receive structured data — JSON payloads, webhooks, API requests, multi-field forms — instead of plain text messages. Define structured triggers for your test cases, and the platform handles the full evaluation lifecycle with full backward compatibility for existing chat-based agents.
  • Zero-Code Trace Ingestion: Convenient trace ingestion in Galtea via an OTeL Collector endpoint. If your service is OTEL-instrumented, the platform will automatically inject W3C traceparent headers on direct inference calls — linking your internal spans to Galtea inference results with no code changes on your side.
  • Automated Actionable Insights: AI-generated insights from your evaluation results, surfacing patterns and recommendations so you can act on findings faster.
